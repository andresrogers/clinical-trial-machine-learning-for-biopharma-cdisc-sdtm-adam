# R Standards Layer Note

This project includes a **thin reference layer** in R for standards-facing credibility.

## Why R is included
- Biopharma standards workflows commonly include R-based derivation examples.
- The R layer demonstrates compatible handling of clinical-style structures while keeping core pipeline ownership in Python.

## Scope boundary
- R is used for a narrow reference path (RDA export support and reference ADSL/ADAE derivations).
- Python remains primary for orchestration, contracts, harmonization, synthetic bridge generation, ADaM-like production outputs, analytics, ML, and reporting.

## Non-goal
This repository does not duplicate the full project pipeline in R.
